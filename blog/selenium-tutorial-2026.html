1. Introduction to Selenium

Selenium WebDriver is the most popular open-source tool for web automation testing. It allows you to automate web browsers and perform various actions like clicking buttons, filling forms, and navigating between pages. In this comprehensive tutorial, we'll cover everything from basic concepts to advanced techniques.

What is Selenium?

Selenium is a suite of tools for automating web browsers. It consists of:

  • Selenium IDE - Record and playback tool for simple scripts
  • Selenium WebDriver - API for browser automation
  • Selenium Grid - Tool for running tests on multiple machines

Why Use Selenium?

  • Open source and free
  • Supports multiple programming languages (Java, Python, C#, Ruby, JavaScript)
  • Cross-browser compatibility (Chrome, Firefox, Safari, Edge)
  • Large community support
  • Integration with various testing frameworks

3. Basic Selenium Commands

Let's start with the fundamental Selenium commands.

Launching Browser

// Java
WebDriver driver = new ChromeDriver();

// Python
from selenium import webdriver
driver = webdriver.Chrome()

Navigating to Websites

// Java
driver.get("https://example.com");
driver.navigate().to("https://example.com");

// Python
driver.get("https://example.com")
driver.navigate().to("https://example.com")

Browser Navigation Commands

// Java
driver.navigate().back();     // Go back
driver.navigate().forward();  // Go forward
driver.navigate().refresh();  // Refresh page

// Python
driver.back()
driver.forward()
driver.refresh()

Getting Page Information

// Java
String title = driver.getTitle();           // Get page title
String url = driver.getCurrentUrl();       // Get current URL
String source = driver.getPageSource();     // Get page source

// Python
title = driver.title
url = driver.current_url
source = driver.page_source

4. Element Locators

Element locators are used to find web elements on a page. Selenium provides 8 different locators:

Types of Locators

  1. ID - Most reliable and fastest
  2. Name - Using name attribute
  3. ClassName - Using CSS class
  4. TagName - Using HTML tag
  5. LinkText - Exact link text
  6. PartialLinkText - Partial link text
  7. CSS Selector - CSS path
  8. XPath - XML path

Locator Examples

// Java
WebElement element = driver.findElement(By.id("username"));
WebElement element = driver.findElement(By.name("email"));
WebElement element = driver.findElement(By.className("btn-primary"));
WebElement element = driver.findElement(By.tagName("input"));
WebElement element = driver.findElement(By.linkText("Login"));
WebElement element = driver.findElement(By.partialLinkText("Log"));
WebElement element = driver.findElement(By.cssSelector("#username"));
WebElement element = driver.findElement(By.xpath("//input[@id='username']"));

// Python
element = driver.find_element(By.ID, "username")
element = driver.find_element(By.NAME, "email")
element = driver.find_element(By.CLASS_NAME, "btn-primary")
element = driver.find_element(By.TAG_NAME, "input")
element = driver.find_element(By.LINK_TEXT, "Login")
element = driver.find_element(By.PARTIAL_LINK_TEXT, "Log")
element = driver.find_element(By.CSS_SELECTOR, "#username")
element = driver.find_element(By.XPATH, "//input[@id='username']")

5. Waits and Synchronization

Waits are crucial for handling dynamic web elements and ensuring test reliability.

Types of Waits

Implicit Wait

// Java
driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);

// Python
driver.implicitly_wait(10)

Explicit Wait

// Java
WebDriverWait wait = new WebDriverWait(driver, 10);
WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("username")));

// Python
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

wait = WebDriverWait(driver, 10)
element = wait.until(EC.visibility_of_element_located((By.ID, "username")))

Fluent Wait

// Java
Wait<WebDriver> wait = new FluentWait<>(driver)
    .withTimeout(30, TimeUnit.SECONDS)
    .pollingEvery(5, TimeUnit.SECONDS)
    .ignoring(NoSuchElementException.class);

WebElement element = wait.until(new Function<WebDriver, WebElement>() {
    public WebElement apply(WebDriver driver) {
        return driver.findElement(By.id("username"));
    }
});

6. Advanced User Interactions

Selenium provides Actions class for handling complex user interactions.

Mouse Actions

// Java
Actions actions = new Actions(driver);
actions.moveToElement(element).click().perform();
actions.contextClick(element).perform();        // Right click
actions.doubleClick(element).perform();        // Double click
actions.dragAndDrop(source, target).perform(); // Drag and drop

// Python
from selenium.webdriver.common.action_chains import ActionChains

actions = ActionChains(driver)
actions.move_to_element(element).click().perform()
actions.context_click(element).perform()
actions.double_click(element).perform()
actions.drag_and_drop(source, target).perform()

Keyboard Actions

// Java
actions.sendKeys(Keys.ENTER).perform();
actions.keyDown(Keys.CONTROL).sendKeys("c").keyUp(Keys.CONTROL).perform();

// Python
actions.send_keys(Keys.ENTER).perform()
actions.key_down(Keys.CONTROL).send_keys("c").key_up(Keys.CONTROL).perform()

7. Working with Frames and Windows

Handling Frames

// Java
driver.switchTo().frame(0);                    // By index
driver.switchTo().frame("frameName");            // By name
driver.switchTo().frame(element);                // By element
driver.switchTo().defaultContent();              // Exit frame

// Python
driver.switch_to.frame(0)
driver.switch_to.frame("frameName")
driver.switch_to.frame(element)
driver.switch_to.default_content()

Handling Windows

// Java
String parentWindow = driver.getWindowHandle();
Set<String> allWindows = driver.getWindowHandles();
for (String window : allWindows) {
    if (!window.equals(parentWindow)) {
        driver.switchTo().window(window);
        break;
    }
}

// Python
parent_window = driver.current_window_handle
all_windows = driver.window_handles
for window in all_windows:
    if window != parent_window:
        driver.switch_to.window(window)
        break

8. Page Object Model (POM)

Page Object Model is a design pattern that makes test automation more maintainable and readable.

Example Page Class

// Java
public class LoginPage {
    private WebDriver driver;
    
    @FindBy(id = "username")
    private WebElement usernameField;
    
    @FindBy(id = "password")
    private WebElement passwordField;
    
    @FindBy(id = "login-btn")
    private WebElement loginButton;
    
    public LoginPage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }
    
    public void login(String username, String password) {
        usernameField.sendKeys(username);
        passwordField.sendKeys(password);
        loginButton.click();
    }
}

9. Best Practices

Code Organization

  • Use Page Object Model
  • Follow proper naming conventions
  • Separate test data from test logic
  • Use configuration files for environment settings

Test Reliability

  • Use explicit waits instead of Thread.sleep()
  • Handle dynamic elements properly
  • Implement proper error handling
  • Use proper assertions

Performance Optimization

  • Reuse browser instances when possible
  • Clean up resources after tests
  • Use parallel execution for faster test runs
  • Optimize locator strategies
